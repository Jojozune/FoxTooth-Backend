Core Components

Authentication Service

Manages user registration, login, and token lifecycle. Implements multi-token system with access/refresh tokens, remember-me functionality, and admin token support.

Player Management

Handles player profiles, statistics, preferences, and online status. Tracks active sessions and maintains player presence data.

Invite System

Real-time invite notifications via WebSocket. Supports invite acceptance/rejection with automatic session creation and automatic cleanup.

Game Session Manager

Creates and manages multiplayer game sessions. Handles host transfer, automatic cleanup, session persistence, and concurrent sessions per player.

Friend Management

Implements friend requests, friend lists, and friend-based grouping. Supports blocking and pending relationship states.

Game Server Registry

Tracks available game servers, their capacity, and current load. Enables dynamic server registration and health checking.

Prerequisites

1 Node.js 16+ - Install from nodejs.org
2 MySQL 5.7+ - Local or remote database instance
3 npm or yarn - Package manager (included with Node.js)

Installation

1 Clone the repository
git clone https://github.com/JojoNoyes/FoxTooth-Backend.git cd FoxTooth-Backend
2 Install dependencies
npm install
3 Configure environment
cp .env.example .env # Edit .env with your database credentials and configuration
4 Run database migrations
npm run migrate
5 Start the server
npm start # Server will start on http://localhost:3000

ğŸ“¡ API Overview

Project Fox Tooth exposes a comprehensive REST API for all backend operations:

Authentication

  • POST /api/auth/register
  • POST /api/auth/login
  • POST /api/auth/refresh
  • POST /api/auth/logout

Players

  • GET /api/players/:id
  • PUT /api/players/:id
  • GET /api/players/status/:id

Invites

  • POST /api/invites/send
  • GET /api/invites/pending
  • POST /api/invites/accept
  • POST /api/invites/reject

Sessions

  • POST /api/sessions/create
  • GET /api/sessions/:id
  • PUT /api/sessions/:id
  • DELETE /api/sessions/:id

Friends

  • POST /api/friends/request
  • GET /api/friends/list
  • POST /api/friends/accept
  • POST /api/friends/block

Servers

  • POST /api/servers/register
  • GET /api/servers/list
  • GET /api/servers/:id

WebSocket Events

Real-time events are delivered via WebSocket connection:
